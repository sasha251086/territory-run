# Territory Run — Предложения по улучшению игры

> Анализ игровой механики и рекомендации на основе изучения кодовой базы.

---

## Контекст

Игра: GPS-бег → влияние на H3-клетки → захват территорий.
- Влияние накапливается до 100/клетку (+1 за посещение)
- Затухает на 2%/день
- Удаляется через 14 дней без активности
- Владелец клетки = игрок с наибольшим влиянием
- Текущий попап клетки показывает только владельца и его влияние — игрок не видит своё место в «борьбе» за клетку

---

## Высокий приоритет

### 1. Прогресс захвата клетки
**Усилие:** мало (1–2 дня)

При клике на клетку показывать позицию текущего игрока: его влияние, место в рейтинге клетки и сколько не хватает до захвата.

**Детали:**
1. Попап клетки сейчас: только владелец + его влияние.
2. Добавить: ваше влияние на клетке (если есть), ваше место (#2 из 5), прогресс-бар до лидера.
3. Формула: нужно пробежек = ceil((leader_influence - your_influence) / influence_per_run). Легко считается на фронте.
4. Если вы уже лидер — показать «Вы владелец» + отрыв от ближайшего конкурента.
5. Endpoint /map/cells уже возвращает ownerId/ownerInfluence; нужно добавить поле myInfluence в ответ.

**Файлы:** `apps/api/src/map/map.service.ts`, `apps/web/src/pages/MapPage.tsx`

---

### 2. Мини-рейтинг клетки (топ-5)
**Усилие:** мало (1–2 дня)

Расширить попап клетки: показывать топ-5 игроков с их влиянием, а не только владельца.

**Детали:**
1. GET /map/cells сейчас отдаёт только топ-1. Нужен эндпоинт GET /map/cells/:h3Index/players.
2. Показывать список: 1. Владелец 87 (+13 до захвата), 2. Вы 74 (...), 3. ...
3. Это даёт «тактическую карту»: игрок видит где он близок к победе, а где отстаёт.

**Файлы:** `apps/api/src/map/map.controller.ts`, `apps/api/src/map/map.service.ts`

---

### 3. Предупреждения о затухании влияния
**Усилие:** мало (1 день)

Визуально выделять на карте клетки, которые игрок скоро потеряет из-за decay или бездействия.

**Детали:**
1. Decay: 2%/день, удаление после 14 дней бездействия. Игрок об этом не знает.
2. На карте: для «моих» клеток менять оттенок зелёного на оранжевый/красный если lastActivityAt > 7 дней.
3. В HUD: добавить счётчик «X клеток под угрозой» (возраст > 10 дней).
4. Опционально: push-уведомление «Кто-то близко к захвату твоей клетки».

**Файлы:** `apps/web/src/pages/MapPage.tsx`, `apps/api/src/map/map.service.ts`

## Средний приоритет

### 4. «Что захватить?» — подсказки рядом
**Усилие:** средне

Кнопка «Найти цели» — показывает ближайшие клетки где у игрока есть шанс захвата за 1–3 пробежки.

**Детали:**
1. Алгоритм: взять клетки в радиусе 2 км, где gap (leader - you) < 15 (1–2 пробежки).
2. Подсветить их на карте отдельным цветом (например, жёлтым).
3. Дать список в виде «3 клетки рядом с вами, нужна 1 пробежка».
4. Это решает главный вопрос нового игрока: куда бежать?

**Файлы:** `apps/api/src/map/map.service.ts`, `apps/web/src/pages/MapPage.tsx`

---

### 5. Районы в интерфейсе
**Усилие:** средне

API районов уже есть (GET /districts/:id), но UI не показывает районы на карте. Добавить границы и короля.

**Детали:**
1. На карте: тонкие границы полигонов районов (GeoJSON уже есть в БД).
2. При клике на район: попап с текущим королём, прогресс-бар контроля (ваш % / 60% порог).
3. В HUD: индикатор «Вы контролируете N% района X».
4. Это добавляет мета-игру и долгосрочную мотивацию.

**Файлы:** `apps/api/src/districts/district.service.ts`, `apps/web/src/pages/MapPage.tsx`

---

### 6. Стрики активности и бонусы
**Усилие:** средне

Давать бонус к влиянию за последовательные дни бега. Стрик 7 дней = +10% влияния.

**Детали:**
1. Добавить поле currentStreak в UserStats (считается при processActivity).
2. Множитель: 1–3 дня = 1.0x, 4–6 = 1.1x, 7+ = 1.2x, 14+ = 1.3x.
3. Показывать стрик в профиле и в HUD на карте.
4. Стрик сбрасывается если нет активности 2 дня (а не 1 — не штрафовать за пропуск выходных).

**Файлы:** `apps/api/src/territories/influence.service.ts`, `apps/api/src/common/constants.ts`, `apps/api/prisma/schema.prisma`

---

### 7. Система соперников
**Усилие:** средне

Добавить возможность «следить» за соперником: видеть его клетки на карте другим цветом.

**Детали:**
1. Профиль → «Следить» за игроком → его клетки на карте выделяются особым цветом.
2. Feed: показывать события отслеживаемых игроков в отдельном разделе.
3. Push: «Иван захватил клетку рядом с тобой».
4. Максимум 3 соперника — не перегружать интерфейс.

**Файлы:** `apps/api/prisma/schema.prisma`, `apps/web/src/pages/MapPage.tsx`

---

### 8. Предпросмотр захваченных клеток после пробежки
**Усилие:** мало

После обработки активности показывать анимацию: карта «вспыхивает» захваченными клетками.

**Детали:**
1. После activity_completed: открыть карту с центром на треке, анимировать зелёным новые клетки.
2. Показать итог: +N клеток, +M влияния, X клеток захвачено впервые.
3. Кнопка «Поделиться» — скриншот карты с маршрутом.

**Файлы:** `apps/web/src/pages/ActivitiesPage.tsx`, `apps/web/src/pages/MapPage.tsx`

## Низкий приоритет

### 9. История владения клеткой
**Усилие:** много

Показывать кто владел клеткой раньше и сколько раз она меняла владельца.

**Детали:**
1. Добавить CellHistory (h3Index, fromUserId, toUserId, changedAt) в schema.prisma.
2. Писать запись при каждом recalculateOwners() при смене владельца.
3. В попапе клетки — раздел «История»: последние 3 смены владельца.

**Файлы:** `apps/api/prisma/schema.prisma`, `apps/api/src/territories/ownership.service.ts`

---

### 10. Понятное объяснение ошибок античита
**Усилие:** мало (полдня)

Сейчас если пробежка отклонена — в UI просто «Ошибка». Нужно объяснять причину и давать инструкцию.

**Детали:**
1. SPEED_EXCEEDED: «GPS-трек содержит нереалистичные скорости. Возможно, вы ехали на велосипеде. Нажмите «Пересчитать» для ручной проверки.»
2. GPS_ANOMALY: «Трек содержит пропуски GPS. Проверьте настройки приложения.»
3. Кнопка «Пересчитать» уже есть в API (reprocess-failed), нужно добавить в UI.

**Файлы:** `apps/web/src/pages/ActivitiesPage.tsx`

## С чего начать

Три быстрых улучшения с максимальным эффектом:

1. **Прогресс захвата клетки в попапе** — 1–2 дня, меняет восприятие игры
2. **Предупреждения о затухании на карте** — 1 день, удерживает игроков
3. **Понятные ошибки античита** — полдня, снижает фрустрацию новичков
