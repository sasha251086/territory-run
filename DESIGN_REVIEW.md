# Дизайн-ревью: Territory Run
**Автор оценки:** AI Design Critic  
**Дата:** 24 июня 2026  
**Объект:** сравнение монохромного мокапа (вложения) с текущим тёмным neon-стилем проекта

---

## 1. Что такое монохромный стиль из мокапов

Мокапы демонстрируют **строгий Swiss-style UI** — белый фон, серая шкала от #F5F5F5 до #1A1A1A, нулевые цвета. Все сигналы строятся исключительно через:

- **Тип и насыщенность шрифта** — заголовки жирные, данные моноширинные, подписи светлые
- **Серые уровни** — фон карточки, рамки, состояния hover/active различаются оттенками серого
- **Один акцентный сигнал** — кнопка действия на всю ширину с инвертированным фоном (чёрное на белом)
- **Высота блоков** — иерархия через размер, а не цвет

Референсы этого подхода: ранний Linear, Notion, Bear Notes, монохромный Strava UI до 2020.

---

## 2. Текущий стиль проекта: Neon Dark Gaming

Текущий `index.css` реализует **gaming HUD-эстетику** с насыщенной палитрой:

```
--bg: #07111f          // глубокий ночной синий
--green: #8dff42       // кислотный lime-зелёный (основной акцент)
--blue: #38bdf8        // cyan
--purple: #b15cff      // фиолетовый
--orange: #fb923c      // предупреждение / стрик
--danger: #fb7185      // ошибка
```

Плюс: `backdrop-filter: blur`, radial-gradient фоны, `drop-shadow` на клетках карты, `text-shadow` на цифрах, анимация decay-pulse на опасных клетках.

## 3. Оценка монохромного подхода для этого продукта

### 3.1 Сильные стороны монохрома

**Читаемость данных.** В мокапах список пробежек, рейтинг, лента — всё воспринимается за один проход. Нет «конкуренции» между зелёными цифрами, оранжевыми стриками и пурпурными рангами. Пользователь сразу видит структуру, а не декор.

**Низкий когнитивный шум.** Монохром снижает нагрузку при частом использовании. Приложение открывают после пробежки, когда человек запыхался и ещё «в теле» — не хочется разбирать световое шоу.

**Универсальность освещения.** На ярком солнце тёмный neon теряет контраст. Светлый монохром работает в любых условиях без перенастройки.

**Профессиональный сигнал.** Минимализм транслирует зрелость продукта. Пользователи воспринимают его серьёзнее, меньше ощущение «очередной геймифицированной поделки».

### 3.2 Слабые стороны монохрома для Territory Run конкретно

**Карта теряет смысл.** Ключевой экран — гексагональная карта территорий. Монохром не позволяет различить «моя клетка / вражеская / нейтральная / в зоне риска» без цвета. В мокапе (экран 2) клетки различаются только оттенками серого — на маленьком экране это превращается в кашу.

**Игровые события не работают без цвета.** «+14 клеток», «захват у rival1», «потеря» — в мокапе это серые бейджи, которые почти неотличимы по срочности. Красный «потеря» должен бить по нерву, иначе игровая мотивация слабеет.

**Нет ощущения прогресса.** Рейтинговый подиум (#1, #2, #3) в монохроме — три серых прямоугольника. В текущем стиле первый светится lime-зелёным, второй — cyan, третий — purple. Это иерархия, которую видно мгновенно.

**Бренд растворяется.** Territory Run — это присвоение города через движение, агрессия и азарт. Монохромный Swiss-стиль ассоциируется с productivity-приложениями, не с захватом территорий.

## 4. Что реально не так с текущим neon-стилем

Проблема не в том, что стиль тёмный или цветной. Проблема в **исполнении конкретных деталей**:

### 4.1 Слишком много акцентов одновременно
В одной карточке могут одновременно светиться:
- `--green` текст с `text-shadow`  
- `--blue` рамка  
- `--orange` стрик-бейдж  
- `backdrop-filter: blur` фон  

Это не иерархия — это хаос. Каждый элемент кричит одинаково громко.

### 4.2 Lime-зелёный (#8dff42) — неудачный выбор основного акцента
Этот оттенок агрессивен на тёмном фоне и вызывает усталость глаз при длительном использовании. Он работает как единичный spark-эффект (например, только для числа клеток), но не как системный цвет для кнопок, рамок, ранков и прогресс-баров одновременно.

### 4.3 Шрифтовой хаос
Три шрифтовых семьи одновременно: `Outfit`, `Inter`, `JetBrains Mono`. Это оправдано только при чёткой роли каждого. Сейчас `JetBrains Mono` используется для цифр статистики — это правильно. Но `Outfit` и `Inter` смешиваются без системы.

### 4.4 Radius-инфляция
`border-radius: 999px` на pill-чипах, `28px` на картах, `24px` на карточках, `18px` на элементах списка — слишком много разных радиусов без единой сетки. Создаёт визуальную дрожь.

### 4.5 Blur-зависимость
`backdrop-filter: blur(18px)` везде — хедер, навбар, карточки, модалки, bottom-sheet. На слабых Android-устройствах это критически влияет на FPS. И визуально: когда blur везде, он перестаёт нести смысл «слой поверх чего-то важного».

## 5. Рекомендация: третий путь — Structured Dark

Ни чистый монохром, ни текущий neon-overload. Оптимальный подход — **строгая тёмная система с одним семантическим цветом**:

| Роль | Текущее | Рекомендую |
|---|---|---|
| Фон | `#07111f` + radial blobs | `#0A0F1A` — единый, без градиентов |
| Поверхность | `rgba(...,0.78)` blur | `#111827` — непрозрачно, быстро |
| Текст | `#f8fafc` | `#F1F5F9` |
| Мuted | `#a7b4c8` | `#64748B` |
| **Акцент (один!)** | lime `#8dff42` | `#22C55E` — зелёный, но не кислотный |
| Опасность | `#fb7185` | `#EF4444` |
| Предупреждение | `#fb923c` | `#F59E0B` |
| Информация | `#38bdf8` | оставить только для карты |

**Принцип:** цвет только на ключевых сигналах — число клеток, CTA-кнопка, критические события на карте. Всё остальное — серые оттенки.

---

## 6. Что сохранить из мокапов

1. **Тип навигации** — нижний таббар с иконкой + подписью работает лучше текущего (текущий хорош, мокап идентичен)
2. **Плотность рейтинга** — три колонки (место / имя / очки + кнопка) в мокапе читаются чище
3. **Post-run экран (2A)** — три крупных числа в центре, затем действие «На карту!». Это верная структура, применить к текущему `RunCelebrationOverlay`
4. **Детали клетки (2Б)** — прогресс-бар «до захвата», топ игроков, история. Хорошая IA, стоит имплементировать в `CellBottomSheet`
5. **Профиль (экран 6)** — компактная сетка 2×2 для метрик (клетки / влияние / дистанция / пробежки) лучше текущего вертикального списка

## 7. Итог

| | Монохромный мокап | Текущий neon | Recommended Dark |
|---|---|---|---|
| Читаемость текста | ✅ Отлично | ⚠️ Средне | ✅ Отлично |
| Карта территорий | ❌ Плохо | ✅ Хорошо | ✅ Хорошо |
| Игровые эмоции | ❌ Нейтрально | ✅ Есть | ✅ Есть |
| Производительность | ✅ Быстро | ⚠️ Blur-нагрузка | ✅ Быстро |
| Брендинг | ⚠️ Productivity-стиль | ✅ Gaming | ✅ Gaming |
| Долгое использование | ✅ Комфортно | ⚠️ Утомляет | ✅ Комфортно |
| Доступность (a11y) | ✅ Высокий контраст | ⚠️ Есть проблемы | ✅ Достаточно |

**Вывод:** монохромный стиль отлично решает проблему читаемости, но убивает игровую душу продукта. Текущий стиль имеет правильный характер, но перегружен. Правильный путь — взять информационную архитектуру из мокапов, сохранить тёмную тему и снизить количество одновременно активных акцентных цветов с 5 до 1–2.
