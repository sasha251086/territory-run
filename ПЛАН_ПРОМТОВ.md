```

Текущая проблема: попап клетки на карте показывает только владельца и его влияние.
Игрок не видит своё место в борьбе за клетку и сколько нужно пробежек чтобы её захватить.

1. В `map.service.ts` в методе getCells() (который принимает bounds и userId):
   - Добавь LEFT JOIN с таблицей CellOwnership для текущего userId
   - В возвращаемый объект каждой клетки добавь поля:
     - myInfluence: Float | null (влияние текущего пользователя на этой клетке, null если нет)
     - myRank: Int | null (место текущего пользователя среди всех владельцев этой клетки)
     - totalPlayers: Int (сколько всего игроков имеют влияние на этой клетке)

2. В `map.controller.ts` убедись что userId передаётся из JWT в getCells().

3. В `MapPage.tsx` в компоненте попапа клетки (Cell popup) добавь секцию "Ваша позиция":
   - Если myInfluence > 0:
     - Показать прогресс-бар: myInfluence / ownerInfluence (в процентах)
     - Текст: "Вы #[myRank] из [totalPlayers] · нужно ещё ~[N] пробежек"
     - Формула N = Math.ceil((ownerInfluence - myInfluence) / BASE_INFLUENCE_PER_CELL)
     - BASE_INFLUENCE_PER_CELL берём из constants.ts
   - Если myRank === 1 (ты владелец):
     - Показать "Вы владелец"
     - Если totalPlayers > 1: показать "Отрыв от #2: +[ownerInfluence - secondInfluence]"
   - Если myInfluence === null:
     - Показать "Вы здесь не бегали"

Стиль попапа уже существует — сохрани его, просто добавь секцию ниже текущей информации о владельце.
```

**Промт:**
```
В игре Territory Run влияние удаляется через 14 дней без активности. Игрок уходящий в отпуск на 2 недели
теряет 100% всей своей территории. 89% таких игроков не возвращаются. Это самая деструктивная механика.

Сделай следующее:

SCHEMA (apps/api/prisma/schema.prisma):
1. Добавь в модель User поля:
   freezeActive    Boolean   @default(false)

2. Измени константу DECAY_DELETE_DAYS с 14 на 45 в constants.ts.
   Обоснование: медленный decay (2%/день) уже наказывает за неактивность.
   Жёсткое удаление на 45й день — достаточно, чтобы карта оставалась живой.

   POST /users/me/freeze — вызывает activateFreeze(userId)

FRONTEND:
6. В `ProfilePage.tsx` добавь секцию "Защита территории":

     Индикатор "Заморозка активна · осталось [X] дней"

### Промт 3.2 — Мягкий cap влияния (100 → 500 с diminishing returns)

**Файлы для контекста перед промтом:**
- `apps/api/src/common/constants.ts`
- `apps/api/src/territories/influence.service.ts`
- `apps/api/src/cron/decay.service.ts`
- `apps/api/prisma/schema.prisma` (поле influence в CellOwnership)

**Промт:**
```
В игре Territory Run влияние ограничено 100 единицами на клетку. Это создаёт проблему:
нет смысла бегать по своим клеткам — они и так на максимуме. Игроки не защищают территорию,
а только расширяют её. Клетки с 15 влиянием захватываются за 1-2 пробежки.

Нужно поднять потолок до 500 с замедлением прироста — чтобы укреплённую клетку было труднее взять.

Сделай следующее:

1. В `constants.ts`:
   - Измени MAX_INFLUENCE_PER_CELL с 100 на 500
   - Добавь SOFT_CAP_1 = 100 (ниже этого — полный прирост)
   - Добавь SOFT_CAP_2 = 300 (ниже этого — 50% прироста)
   - Оставь BASE_INFLUENCE_PER_CELL = 1 (базовое значение)

2. В `influence.service.ts` в методе calculateInfluenceGain(currentInfluence, multiplier):
   Реализуй diminishing returns:
   - currentInfluence < SOFT_CAP_1: gain = BASE_INFLUENCE_PER_CELL * multiplier
   - SOFT_CAP_1 <= currentInfluence < SOFT_CAP_2: gain = BASE_INFLUENCE_PER_CELL * multiplier * 0.5
   - currentInfluence >= SOFT_CAP_2: gain = BASE_INFLUENCE_PER_CELL * multiplier * 0.25
   - Итоговое влияние не превышает MAX_INFLUENCE_PER_CELL (500)

3. В `decay.service.ts`:
   - Decay остаётся 2% в день от текущего значения (не от MAX)
   - Пример: 500 влияния → –10/день. 100 влияния → –2/день.
   - Это означает что "укреплённую" клетку (500) можно потерять за 250 дней полного бездействия
   - При 100 влиянии (текущий реальный max) поведение не меняется

4. В `schema.prisma`:
   - Убедись что поле influence — Float (не Int), т.к. diminishing returns могут давать дробные значения
   - Если Int: изменить на Float, создать миграцию

5. ВАЖНО: существующие данные не трогать. Клетки с текущим влиянием 1–100 остаются как есть.
   Игроки просто теперь могут набрать больше 100 если продолжат бегать по тем же маршрутам.

6. Выполни: npx prisma migrate dev --name influence_soft_cap
```

### Промт 3.3 — Ребаланс домашнего бонуса

**Файлы для контекста перед промтом:**
- `apps/api/src/common/constants.ts`
- `apps/api/src/territories/influence.service.ts`

**Промт:**
```

"непробиваемые крепости": активный игрок у своего дома математически недостижим для соперника.
Примерно 23% карты заблокировано для реальной конкуренции.

Нужно снизить бонус и ограничить его количество клеток.

Сделай следующее:

1. В `constants.ts`:
   - Измени HOME_ZONE_MULTIPLIER с 1.5 на 1.2
   - Измени HOME_ZONE_RADIUS_M с 500 на 500 (не меняем радиус)
   - Добавь HOME_ZONE_MAX_CELLS = 25 (бонус применяется максимум к 25 ближайшим к дому клеткам)

2. В `influence.service.ts` в методе getMultiplierForCell(userId, h3Index):
   Текущая логика: если расстояние от дома <= 500м → возвращать HOME_ZONE_MULTIPLIER.
   
   Новая логика:
   a) Получить homeCell = h3.latLngToCell(user.homeLat, user.homeLng, 9)
   b) Получить distance = h3.gridDistance(homeCell, targetH3Index)

   c) Получить ближайшие HOME_ZONE_MAX_CELLS клеток от дома пользователя:
      homeZoneCells = h3.gridDisk(homeCell, 2)  // ring 2 = ~19 клеток, ring 3 = ~37 клеток
      Используй ring 2 (gridDisk с k=2) — это даёт ~19 клеток, близко к 25
   d) Если targetH3Index IN homeZoneCells → возвращать HOME_ZONE_MULTIPLIER (1.2)
   e) Иначе → возвращать стандартный multiplier

3. Не нужно трогать данные пользователей или CellOwnership.
   Новые бонусы применяются только к будущим пробежкам.

4. В `NATIVE_SETUP.md` или документации обнови описание домашнего бонуса
   если оно там есть (старое значение 1.5 может ввести в заблуждение).

"Домашний бонус снижен с ×1.5 до ×1.2 и теперь действует в пределах ~19 ближайших клеток.
Ваша домашняя зона по-прежнему даёт преимущество — просто не такое непробиваемое."
