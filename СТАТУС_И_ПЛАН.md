# Territory Run — оценка проекта и план действий

_Обновлено: 22 июня 2026_

Этот файл — короткая сводка «что готово» и пошаговый список «что делать дальше».
Написан простым языком, без жаргона.

---

## 1. Что это за проект

Игра для бегунов. Ты загружаешь пробежку (файл GPX), маршрут превращается
в шестиугольные клетки на карте, клетки приносят «влияние», и территория
становится твоей. Влияние со временем убывает, районом можно «владеть»,
захватив большинство клеток.

- **Сайт (игрок):** https://territory-run-cjoj.onrender.com
- **API (сервер):** https://territory-run-api-erbs.onrender.com

**Вывод:** играбельная версия собрана. Бэкенд и сайт работают, пробежку
можно загрузить файлом, клетки захватываются (проверено: 12 клеток после
загрузки `Утренний_забег.gpx`).

### Про этап 5 (почему «отложено», а не «пропущено»)
Это оптимизация хранения координат в базе. Нужна только когда станет
очень много клеток/пользователей и карта начнёт тормозить. Сейчас всё
считается в коде и работает. Это не дыра — так задумано планом.

## 3. Что исправлено в последней сессии (ВАЖНО: ещё не задеплоено)

В коде сделаны правки, но они **пока не на сайте** — нужен деплой:

1. **Карта рисовала клетки не в том месте** — перепутаны широта/долгота.
   Исправлено.
2. **Клетки пропадали при смене домашней базы** — карта показывала только
   видимую область. Добавлен слой «мои клетки» и кнопка
   **«Показать мои клетки»** (карта сама перелетает к твоим территориям).
3. **Новый серверный метод** `GET /map/cells/mine` — отдаёт все твои
   клетки, где бы они ни были.
4. **Кнопка «Обновить профиль»** раньше молчала. Теперь:
   - «Обновить статистику» — с откликом и сообщением;
   - «Изменить домашнюю базу» — открывает карту для выбора новой точки.
5. **Профиль:** добавлена подсказка про загрузку GPX на вкладке «Пробежки».

> Эти изменения существуют только на твоём компьютере. Пока не сделан
> деплой (шаги ниже), на сайте их не видно.

## 4. Дальнейшие действия — пошагово

### Шаг 1. Проверить, что код собирается
- В папке проекта собрать сервер и сайт локально.
- Команды: `pnpm --filter api build` и `pnpm --filter web build`.
- Цель: убедиться, что нет ошибок до публикации.

### Шаг 2. Сохранить изменения в git (commit)
- Добавить изменённые файлы и сделать commit с понятным описанием,
  например: «fix: отображение клеток на карте, слой мои клетки, правки профиля».
- Файлы, которые менялись:
  - `apps/web/src/pages/MapPage.tsx`
  - `apps/web/src/pages/ProfilePage.tsx`
  - `apps/web/src/pages/OnboardingPage.tsx`
  - `apps/web/src/context/AuthContext.tsx`
  - `apps/web/src/index.css`
  - `apps/api/src/map/map.controller.ts`
  - `apps/api/src/map/map.service.ts`

### Шаг 3. Отправить на GitHub (push)
- `git push` в основную ветку.
- Render следит за репозиторием и сам начнёт пересборку.

### Шаг 4. Дождаться деплоя на Render
- **API** (`territory-run-api`) — пересоберётся из `apps/api`.
- **Сайт** (`territory-run-web`) — пересоберётся из `apps/web`.
- Подождать, пока оба сервиса станут «Live».

### Шаг 5. Проверить на сайте
- Зайти на https://territory-run-cjoj.onrender.com
- Жёстко обновить страницу (Ctrl+Shift+R) — чтобы ушёл старый кэш.
- Вкладка «Карта» → нажать **«Показать мои клетки»** → увидеть зелёные
  шестиугольники в Уфе (там, где был трек).
- Профиль → проверить «Обновить статистику» и «Изменить домашнюю базу».

### Шаг 6. Решить про домашнюю базу
- Если играешь в Уфе — поставить домашнюю базу там (бонус влияния +500 м).
- Если в Риге — загружать пробежки из Риги.
- Старый трек из другого города годится только для теста.

## 5. Что можно сделать дальше (по желанию, не срочно)

В порядке полезности:

1. **Загрузка ZIP из Samsung Health** — чтобы не пользоваться внешними
   конвертерами: игра сама достаёт GPS из архива «Download personal data».
2. **Поддержка формата TCX** (Garmin) — сейчас принимается только GPX.
3. **Показ маршрута пробежки линией** на карте (а не только клетки).
4. **Улучшить онбординг** — мини-инструкция «как получить GPX».
5. **Этап 5 (PostGIS)** — только когда клеток станет десятки тысяч.

---

## 6. Известные внешние ограничения (не баги проекта)

- **Strava как авто-синхронизация** требует платную подписку разработчика
  Strava API — поэтому основной путь сейчас это загрузка GPX-файлом.
- **Apple Health / Health Connect** недоступны из браузера — ограничение
  iOS/Android, не проекта.
- **Samsung Health** в некоторых регионах не показывает «Export GPX» —
  обходные пути описаны ранее (FitoTrack, Strava-экспорт, конвертер из
  personal data).

## 7. Короткий ответ на вопрос «что делать прямо сейчас»

1. Собрать проект (Шаг 1).
2. Commit + push (Шаги 2–3).
3. Дождаться Render (Шаг 4).
4. Проверить кнопку «Показать мои клетки» на карте (Шаг 5).

После этого пропавшие полигоны вернутся, и игра будет полностью рабочей.
